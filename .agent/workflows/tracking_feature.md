# Real-time Shipment Tracking Workflow

This workflow describes how to use the real-time shipment tracking features, including route optimization, automated notifications, and predictive delivery timelines.

## Features

1.  **Real-time Tracking**: Visualize the shipment's live location on an interactive map.
2.  **Optimized Route**: The map displays the optimized route generated by OSRM.
3.  **Predictive Analytics**:
    -   **Predictive ETA**: A dynamic ETA based on current speed and conditions, displayed alongside the original scheduled ETA.
    -   **Delay Risk Analysis**: A risk assessment (High/Medium/Low) based on driver behavior and progress.
4.  **Journey Timeline**: A real-time log of automated notifications and shipment events (e.g., "Out for Delivery", "Delayed").

## User Steps

1.  **Navigate to Shipments Page**: Go to the `/shipments` page in the application.
2.  **Open Tracking Modal**: Click the "Track" button on any active shipment card.
3.  **View Tracking Details**:
    -   **Map**: Observe the vehicle's current position and the blue route line.
    -   **Right Sidebar (Desktop)**: Check the "Delivery Risk Analysis" card and the "Journey Timeline".
    -   **Header**: Compare the "Predictive ETA" with the original time (struck through if different).

## Backend implementation

-   **Route Optimization**: Handled by `routeService.js`.
-   **Predictive Analytics**: Handled by `predictiveAnalyticsService.js`.
-   **Notifications**: Handled by `notificationService.js` and `shipmentController.js`.
-   **Tracking Endpoint**: `GET /api/shipments/:id/tracking` returns `shipment`, `locationHistory`, `delayRisk`, and `recentEvents`.

## Verification

To verify the features:
1.  Open Chrome Developer Tools.
2.  Check the Network tab when opening the modal.
3.  Verify the response from `/tracking` includes `delayRisk` and `recentEvents`.
